# Wooden Ships & Iron Men v1.2b — 表・チャート日本語解説

帆船戦ゲーム開発の参考資料として、Avalon Hill 社『Wooden Ships & Iron Men』トーナメント版ルール v1.2b に含まれる各種表・チャートの構造と使い方を日本語で解説する。各節は原典の表名（英語）と対応し、実装者向けに入力・出力・手順を明示する。

---

## 1. 手番進行 (Advanced Game Sequence of Play)

**目的**: 1 ターン内の処理順序を固定し、同時解決ルール（特に射撃と移動）を正しく扱うための基準。

**構造**: 10 個のフェイズが A～J の順に並ぶ。各フェイズ内の下位手順は「+」で列挙されている。

**判定手順**:
1. **A. Wind Phase（風）**: 2d6 を振り、風の変化を判定（後述の Wind Change Table 参照）。
2. **B. Unfouling Phase（絡み解除）**: 前ターンまで絡んだ艦の解放判定、沈没・爆発ロール、火災ロール。
3. **C. Movement Notation Phase（移動記述）**: 各艦のログに移動経路・修理・錨の上げ下ろし・帆の切替 (FS↔BS) を筆記。
4. **D. Movement Execution Phase（移動実行）**: 記述通り全艦を動かし、その後に drift（漂流）艦を処理。衝突・絡みを解決し、帆切替艦に MIR マーカーを置く。
5. **E. Grappling and Ungrappling Phase（鉤縄）**: 鉤縄の成功／回避／解除判定。
6. **F. Boarding Preparation Phase（乗込準備）**: OBP/DBP/TBP/NBP をログに記述、消火要員も割当。
7. **G. Combat Phase（射撃）**: 全砲撃を同時解決しログに被害記入、クリティカル判定、降伏艦の確認。
8. **H. Melee Phase（白兵）**: 全白兵戦闘を解決。
9. **I. Reload Phase（再装填）**: 空の舷側 1 個を再装填し reload 欄を 1 マス（ダブルショットなら 2 マス）消す。修理完了を宣言しダメージを消去。
10. **J. Full Sail Phase（全帆）**: Movement Notation で指示したとおり FS マーカーを置去／撤去、MIR マーカーを撤去。

**実装メモ**: 射撃が同時解決である点が最重要。ログ記入と処理の分離を徹底するため「プロット」と「解決」を別ターンサブステップとして保持すること。

## 2. 命中決定表 (Hit Determination Table / HDT)

**目的**: 発砲時に使用する「Hit Table の番号（0～10）」を、射程と発砲砲数から決定する。ここで求めた番号と各種修整値の合計が、次段の Hit Table へのインデックスとなる。

**構造**:
- **行**: 射程（ヘクス数）。1, 2, 3, 4, 5–6, 7–10 の 6 段階。
- **列**: 発砲する gun square の数。1–3, 4–6, 7–9, 10–12, 13–15, 16–18, 19–21, 22–24, 25+ の 9 段階。
- **マス値**: 基本 Hit Table 番号（通常砲撃）。カッコ内の数値は **Rake（縦射）** 時の Hit Table 番号。

**判定手順**:
1. 発砲舷の健全な gun square 数を数える。カロネードは射程 2 以下なら加算。
2. ターゲットまでのヘクス距離を測る。
3. 行（射程）×列（砲数）の交点を参照し、通常値を取得。Rake ならカッコ内を使用。
4. 後述の HDT 修整を全て加減算し、最終的な Hit Table 番号を得る。
5. 0 未満なら自動的に外れ。8 を超えたら 8 として扱う（上限）。

**代表値（抜粋）**:
| 砲数 \ 射程 | 1 hex | 2 hex | 3 hex | 4 hex | 5–6 | 7–10 |
| ---: | :---: | :---: | :---: | :---: | :---: | :---: |
| 22–24 門 | 5 (9) | 4 (8) | 4 (8) | 4 (7) | 3 (7) | 2 (6) |
| 13–15 門 | 3 (6) | 3 (6) | 3 (5) | 2 (5) | 2 (5) | 1 (5) |
| 7–9 門 | 2 (4) | 2 (4) | 1 (4) | 0 (3) | 0 (3) | -1 (0) |
| 1–3 門 | 0 (1) | -1 (0) | -2 (-1) | -3 (-2) | -4 (-3) | 自動外れ |

**実装メモ**: 「基本 Hit Table 番号 + 修整 = 最終 Hit Table 番号」。この番号が次の Hit Table #0～#10 の選択キーになる。

## 3. Hit Table 修整表 (Hit Table Modifiers)

**目的**: HDT で求めた基本値に加える修整を提供。

**構造**: 項目別の固定修整。次の 2 カテゴリに分かれる。

### 基本ゲーム修整（Basic Modifiers）
| 条件 | 修整 |
|---|---|
| Stern Rake（艦尾縦射） | +1 |
| Initial Broadside（初回舷側射撃） | +1（両舷それぞれ 1 回のみ） |
| Crew Section Loss（艦員区画喪失／乗込団に使用） | 完全喪失 1 区画につき -1 |

### 上級ゲーム修整（Advanced Modifiers）
| 条件 | 修整 |
|---|---|
| At Full Sail（全帆中） | -1（射撃側が全帆） |
| All Anchor（錨泊中） | +1 |
| Captured Ship（拿捕艦の砲を使用） | -2（艦員喪失修整は無視） |
| Crew Quality Elite | +2 / Crack +1 / Average 0 / Green -1 / Poor -2（砲数に応じて詳細変動あり） |

### 弾種修整（Ammunition Type — Optional）
| 弾種 | 修整 |
|---|---|
| Round Shot（丸弾） | 0（基準） |
| Chain Shot（鎖弾） | +2（ただし H と G 命中は外れ扱い） |
| Double Shot（二重装弾） | +2（射程 3 以内） |
| Grape Shot（ぶどう弾） | 修整後の値＝殺傷艦員数（Hit Table 不使用） |
| Carronade | 射程 2 以内のみ |
| Low Powder（火薬欠乏、リロード尽き） | -2～-1（砲数帯により変動）|

**判定手順**: 適用可能な全修整を累積し、HDT の基本値に加減算してから 0～8 にクリップする。

**実装メモ**: 弾種は装填時（前々ターン）に決定しており、リロード消費量が異なる（ダブルショットは 2 マス消費）。

## 4. 弾種射程表 (Munition Range Table)

**目的**: 各弾種の最大射程と火薬欠乏時の扱いを規定。

**構造**: 行＝弾種／砲種、列＝最大射程（ヘクス）。

**代表値**:
| 弾種 | 最大射程 |
|---|---|
| Round Shot | 10 hex |
| Chain Shot | 4 hex |
| Double Shot | 3 hex |
| Grape Shot | 2 hex |
| Carronade | 2 hex |

**実装メモ**: 射程外での射撃は一切不可。Carronade は弾種に追随（舷側と同じ弾種）する。

## 5. ヒット表 (Hit Tables #0–#10)

**目的**: HDT で得た Hit Table 番号ごとに、1d6 の出目と被害の種類・量を規定する。艦砲の威力段階を離散化したもの。

**構造**:
- テーブルは #0～#10 の 11 枚。番号が大きいほど強力。
- 各テーブルは **Hull（船体）** と **Rigging（帆柱）** の 2 セクションに分かれる。攻撃側はいずれを狙うか発砲前に宣言。
- 各セクションは 1d6 の出目 1～6 に対応するセル 6 個を持つ。

**セル内容の表記法**:
- 数字 + 文字で被害種別と数を示す。文字コードは次の通り。
  - **R** = Rigging square（帆柱マス）
  - **H** = Hull square（船体マス）
  - **G** = Gun square（砲マス、被弾舷側の最寄り区画から）
  - **C** = Crew square（艦員マス、1 区画目から順）
- カッコ内の値 `(…R)` は **Full Sail 状態のターゲットに対する追加 R 被害**（rule 20.1.3）。
- `*` は 6 の目でクリティカル判定の可能性（rule 16.1.2.2）。

**代表例**:
- Table #2, Rigging section: 出目 1=0(R), 2=R, 3=H, 4=R-G, 5=R-C, 6=2R-H-C*(R)
- Table #10, Hull section: 出目 6=7R-H-G-2C*(4R) のような大被害。

**判定手順**:
1. 攻撃側が船体／帆柱のどちらを狙うか宣言。
2. 対応する Hit Table # のセクションを参照。
3. 1d6 を振り、出目の列のセルを読む。
4. 指定された H/R/G/C 個数だけターゲットのログからマスを消去。
5. Full Sail の対象へ Rigging を撃つなら `(…R)` の値を通常被害に加算。
6. 6 の目で `*` があればクリティカル判定へ。

**実装メモ**: セル内容はテキストパース必須。`{hull:3, rigging:0, gun:1, crew:2, fullSailBonusR:1, crit:true}` のような構造体で保持すると扱いやすい。Gun 被害は被弾舷側の近い区画から、Crew は常に第 1 区画から消費する。

## 6. クリティカル命中表 (Critical Hit Table)

**目的**: Hit Table で `*` 付きの 6 が出たときに追加で発生する特殊被害を判定。

**構造**: 2d6 の出目 2～12 を行、被弾部位（Hull または Rigging）を列とする 22 セル。各セルは文字列の効果（独立ルール）で表される。

**判定手順**:
1. Hit Table で 6 を引き、セルに `*` が付いていた場合のみ。
2. 2d6 を振り、被弾セクション（船体か帆柱か）を選び交差セルを読む。
3. セルが「No Effect」なら無効。そうでなければ効果処理を行う。
4. 複数効果は累積し、Combat Phase 終了時の通常被害処理後に適用する。

**代表効果**:
| 出目 | Rigging 側 | Hull 側 |
|---|---|---|
| 2 | 2R + fire check（火災判定） | 2H + 火災+弾薬庫誘爆チェック（1d6 が射程＋4 以下で爆沈、それ以外は 2H 火災） |
| 3 | 3R（マスト落下判定：1d6 ≤ 射撃側 Hit Table # ならマスト倒壊、当該 Rigging セクション全壊） | 3H + 艦員士気崩壊 |
| 4 | 4R（同上マスト判定） | 4H（舵破壊、次ターン回頭不可） |
| 5 | 5R | 5H（水線下損傷、残り全ターン艦員 1 区画を釘付け） |
| 6 | 6R | 6H |
| 7 | 7R | 7H |
| 8 | 8R | 8H |
| 9 | 9R | 9H |
| 10 | 10R | 10H |
| 11 | 11R | 11H |
| 12 | 12R | 12H |

**代表効果の意味**:
- **マスト倒壊**: 対応 Rigging セクションを全消し。
- **水線下損傷**: 1 艦員区画を航海終了まで排水作業に固定。
- **火災**: 毎ターン消火判定。1d6 ≤ 割当艦員区画数なら消火、それ以外は 1H + 1R が追加被害。
- **舵／操舵系破壊**: 次ターン回頭不可。舵破壊は永続的に回頭力 -1（艦尾 Rake の場合は無効）。
- **士気崩壊**: 1d6 + 艦員質値 - 失艦員区画数 が 6 以下なら艦員質が 1 段階低下。

**実装メモ**: 全効果は「Combat Phase の通常被害をログ反映後」に適用。`*` は Table #0–#2 の低威力 Hit Table でも存在するため、6 の目が出た時点で必ず確認する。

## 7. 風変化表群 (Wind Change Tables)

風は 3 つの別表で管理される。

### 7.1 Wind Change Table（風方向変化）
**目的**: 風の方向がどう変わるかを決定。

**構造**: 1d6 の出目 → 効果。
| 出目 | 効果 |
|---|---|
| 1 | 風向き 120 度反時計回り |
| 2 | 60 度反時計回り |
| 3 | 初期方向へ戻る |
| 4 | 変化なし |
| 5 | 60 度時計回り |
| 6 | 120 度時計回り |

### 7.2 Wind Velocity Change Table（風速変化）
**目的**: 風速が 1 段階下がる／変わらない／1 段階上がるか。

**構造**: 1d6 の出目 → 速度変化。
| 出目 | 効果 |
|---|---|
| 1–2 | 風速 -1 |
| 3–4 | 変化なし |
| 5–6 | 風速 +1 |

### 7.3 Changing Wind Number Chart（風変化番号更新）
**目的**: 方向と風速の両方が変化したターンのみ、任意で 4 回目のダイスを振り「次ターン以降の Wind Change Number」を決める。

**構造**: 行＝1d6 の出目（1–6）、列＝既存の新 Wind Change Number 候補。
**代表値**: 出目に応じて新 WCN は 7／9／11 のいずれか、または複数から選択可能な帯になる。

**判定手順**:
1. Wind Phase で 2d6 を振る。
2. 結果＝現在の Wind Change Number なら風変化発生、さもなくば無変化。
3. 変化時は **1d6 で 7.2（速度）**、**1d6 で 7.1（方向）** を判定。
4. 両方変化した場合のみ、希望すれば 4 回目を振って 7.3 を参照し新 WCN を設定。

**実装メモ**: WCN の初期値はシナリオ指定。風速は 0（無風）～7（ハリケーン）の 8 段階。風速 5 以上は Class 1・5・6 の Hit Table -1 など Hit Table にも波及する。

## 8. 絡み関連 4 表 (Fouling / Unfouling / Grappling / Ungrappling Tables)

すべて 2d6 を振って閾値 10+ を基本目標とする系統の表。

### 8.1 Fouling Table（絡み発生）
**目的**: 衝突や移動後に 2 艦が絡まるかの判定。
**構造**: 2d6 結果が 10+ で絡み発生、2–9 で絡まず。12 は常に絡む。
**修整**:
- 全帆中の艦 1 艦につき +1
- いずれかが静止なら -4
- 対象艦が 3 MP 以上で記述されている場合 +1
- 両艦が同盟国 -1
- 艦員質：Green +1／Poor +2／Crack -1／Elite -2

### 8.2 Unfouling Table（絡み解除）
**構造**: 2d6 が 10+ で解除、2–9 で継続。12 は常に解除。
**修整**:
- 全帆中 -1
- 白兵進行中 -6
- Rigging 1 マス消しで +1（意図的損傷による脱出試行）
- Elite +2／Crack +1

### 8.3 Grappling Table（鉤縄成功）
**構造**: 2d6 が 10+ で成功。12 は常に成功。
**修整**:
- 風速 6 時 -2／風速 5 時 -1／風速 1 時 +1
- 対象艦静止 1 艦あたり +1
- 対象艦の移動 MP（1 超の分）あたり -1
- 両艦同盟 +4
- 艦員質差 ±1 ごとに ±1

### 8.4 Ungrappling Table（鉤縄解除）
**構造**: 2d6 が 10+ で解除。12 は常に解除。同盟艦なら自動解除。
**修整**:
- 白兵進行中 -6／非進行時 +1
- Elite +2／Crack +1
- 風速 6 +1

**判定手順**: 2d6 を振り、修整後の合計を閾値と比較。出目そのものが 12 なら修整無視で自動成功／自動解除。

**実装メモ**: 「12 は常に○○」は修整を無視した帯外し成功を表す。ダイスロール生値も保持して分岐させる必要がある。

## 9. 艦員白兵強度表 (Crew Melee Strength Table)

**目的**: 乗込戦闘における 1 艦員区画の火力点（melee factor）を決定。

**構造**: 行＝艦員質、列＝乗込団タイプ。
| 艦員質 | DBP（防御乗込団） | DBP*（Rake を受けた DBP） | OBP（攻撃乗込団） |
|---|---|---|---|
| Elite | 7 | — | 6 |
| Crack | 6 | — | 5 |
| Average | 5 | — | 4 |
| Green | 4 | — | 3 |
| Poor | 3 | — | 2 |

※ DBP\*（被 Rake 中）の値は DBP から -1 相当を意味する欄。実際の原票に従うこと。

**判定手順**:
1. 乗込団に投入された艦員区画数を数える。
2. 表から 1 区画あたりの値を取得。
3. 「区画数 × 区画当たり値」＝ **Total Melee Strength (TMS)**。

**実装メモ**: OBP は DBP より 1 ポイント低い。DBP の防御ボーナスは「攻撃艦が Rake されていない限り」有効。

---

## 10. 白兵解決表 (Melee Resolution Table)

**目的**: TMS と 1d6 から「倒す敵艦員区画数」を算出。

**構造**: 行＝1d6（1–2、3–4、5–6 の 3 帯）、列＝TMS の帯（1–10, 11–20, 21–30, 31–40, 41–50, 51–60, 61–70, 71–80, 81+ の 9 帯）。マス値は敵艦員区画の喪失数。

**代表値**:
| 1d6 \ TMS | 1–10 | 11–20 | 21–30 | 31–40 | 41–50 | 51–60 | 61–70 | 71–80 | 81+ |
| ---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| 1–2 | 0 | 1 | 1 | 1 | 1 | 2 | 2 | 2 | 3 |
| 3–4 | 0 | 1 | 1 | 2 | 2 | 2 | 3 | 3 | 4 |
| 5–6 | 0 | 1 | 2 | 2 | 3 | 3 | 4 | 4 | 5 |

**判定手順**:
1. 両軍それぞれ TMS を算出。
2. 各軍が同時に 1d6。
3. 表を引いて相手の艦員区画を消去。
4. 1 ターンに最大 3 ラウンド繰返し、決着しなければ次ターン継続。

**実装メモ**: 白兵は必ず同時解決。勝利条件は ① 降伏 ② 絡み解除 ③ 相互合意離脱 のいずれか。

---

## 11. 破壊船体表 (Destroyed Hull Table)

**目的**: Hull の全マスが消えた艦の結末を決定。

**構造**: 1d6 の出目 → 結果。
| 出目 | 結果 |
|---|---|
| 1–4 | 旗を降ろして降伏（Striking） |
| 5 | 沈没の恐れ（Ship may sink\*） |
| 6 | 爆発の恐れ（Ship may explode\*） |

**判定手順**:
1. 最後の Hull マスが消えた瞬間にロール。
2. 結果 5／6 の場合、以降のターン Unfouling Phase で再ロール。「6」が出たら沈没／爆発確定（rule 24.3）。

**実装メモ**: 降伏（Striking）は操船・砲撃・白兵不可。拿捕可能。

## 12. 艦船ログ (Ship's Log)

**目的**: 各艦の現在状態（体力・艦員・砲・帆柱・装填・移動記述）を記録する管理シート。

**構造**: 改訂版ログ (Revised Ship's Log) は 31 個の枠で構成。主な区画は以下。

| 番号 | 部位 | 用途 |
|---|---|---|
| 1 | Ship's Name | 艦名 |
| 2 | Identification # | 駒の 4 桁番号 |
| 3 | Crew Quality | 艦員質（＋勝利点メモ可） |
| 4 | Class Rating | クラス（例: SOL 1 – 120 guns） |
| 5–8 | Wind Attitude A–D | 風向別の Battle Sail 移動力／(カッコ内)Full Sail 移動力 |
| 9 | Wind Change Number | 現在の WCN |
| 10 | Wind Direction | 現在風向 |
| 11–14 | Rigging Sections 1–4 | 帆柱区画（最大 4 セクション。**損失は最大番号から消去**） |
| 15–17 | Crew Sections 1–3 | 艦員区画（**損失は最小番号から消去**） |
| 18–21 | Starboard Broadside（Stern Carronade／Stern Gun／Bow Gun／Bow Carronade） | 右舷砲・カロネード |
| 22 | Turning Ability | 回頭力（カウンター初期値から消込） |
| 23 | Hull | 船体 30 マス（未使用は消込） |
| 24 | Waterline Damage | 水線下被弾マーカー用 W の丸 |
| 25 | Anchor Cables | 錨索 |
| 26–29 | Port Broadside | 左舷砲・カロネード（18–21 と対称） |
| 30 | Reloads | 再装填残量（初期 30、0 で Low Powder 状態） |
| 31 | Depth (ft.) | 喫水（座礁判定用） |

**判定手順（利用方法）**:
1. シナリオ／Ship Specification Sheet から該当艦を取得しログに転記。
2. 使用しないマスは事前に消し込む（例: Rigging 7 マスしか持たない艦は 8–10 を消す）。
3. 戦闘中は被弾ごとに該当マスへ「X」を記入。
4. 艦員は最小番号から、帆柱は最大番号から消すルールを厳守。
5. Reload 消費は最大番号マスから消す（30→29→…）。0 到達で Low Powder。

**実装メモ**:
- Rigging の消去順（高番優先）と Crew の消去順（低番優先）は対称関係で実装ミスが出やすい。
- Carronade は Gun 区画とは別マス（点線枠）で管理し、射程 2 以下でのみ砲数に加算。
- Broadside は Bow／Stern に二分されており、被弾時は「射撃軸に近いセクション」から消去（rule 24.1.7）。

## 実装上の処理順序（要約）

射撃 1 回分の完全な処理チェーン:

1. **舷側・弾種決定** — Port/Starboard、Round/Chain/Double/Grape/Carronade を宣言。
2. **砲数集計** — 健全 gun square 数を集計（射程 2 以下なら Carronade 加算）。
3. **HDT 引き** — 射程×砲数で基本 Hit Table 番号を取得（Rake ならカッコ内）。
4. **修整適用** — Crew Quality／Initial Broadside／Crew Loss／Full Sail／Anchor／Captured／弾種／Low Powder／風速効果を累積加算。
5. **Hit Table # 確定** — 0–8 にクリップ。0 未満は自動外れ。
6. **被弾面選択** — Hull または Rigging を射手が宣言。
7. **Hit Table #n 引き** — 1d6 → セル → R/H/G/C 数を取得。Full Sail 対象なら追加 R を合算。
8. **クリティカル判定** — 6 の目かつ `*` 付きなら 2d6 で Critical Hit Table 引き。
9. **被害記入** — Hull／Crew（最小番号から）／Rigging（最大番号から）／Gun（近いセクションから）をログへ記入。
10. **副次効果処理** — 水線、火災、マスト倒壊、降伏、Hull 消尽時は Destroyed Hull Table を引く。

全ての射撃は **同時解決** なので、一括でプロットしてからダイスを振る運用を前提に実装する。
