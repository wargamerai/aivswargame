# Wooden Ships & Iron Men v1.2b ゲーム仕組み 日本語解説

本ドキュメントは、帆船戦術シミュレーション「Wooden Ships & Iron Men (WS&IM) v1.2b Tournament Edition」のルール文書を、**帆船戦ゲームの実装参考資料**として日本語で要約・解説したものです。原文の翻訳ではなく、各ルールの**仕組み・設計意図・判定の流れ**を整理しています。

---

## 1. ゲーム概要 (Game Overview)

### 1.1 舞台と規模
- 対象時代: 1776〜1814年 (米国独立戦争〜ナポレオン戦争期)
- 戦列艦 (ship-of-the-line, SOL)、フリゲート (frigate)、ブリッグ (brig) などの帆船艦隊戦術を再現
- 1ユニット = 1隻、ヘックスマップ上で2ヘクスを占有 (bow hex(艦首ヘックス) と stern hex(艦尾ヘックス))
- 1ターン = 実時間約3分
- プレイヤーは1隻〜艦隊までを指揮

### 1.2 艦級分類 (Ship Classes)
艦は7段階のクラスで分類され、クラス番号が小さいほど大型:

| クラス | 種別 |
|---|---|
| 1-2 | Ship-of-the-line (戦列艦 SOL) |
| 3 | Frigate (フリゲート F) |
| 4 | Frigate / Corvette (コルベット C) |
| 5 | Brig / Sloop / Schooner (ブリッグ、スループ、スクーナー) |
| 6 | Gondola / Galley / Radeau |
| 7 | Gunboat (砲艇) |

その他に Privateer(私掠船)、Merchantman(商船)、Fireship(火船)、Bomb Ketch(臼砲船) が存在。

### 1.3 乗員品質 (Crew Quality)
- **Elite → Crack → Average → Green → Poor** の5段階
- 命中判定、白兵戦、接舷、離舷などすべての判定に影響する中核パラメータ

### 1.4 ログシート方式
各艦は「ログ (log)」と呼ばれる管理シートを持ち、以下を記録:
- 艦名、ID、艦級、乗員品質
- 風向4態勢 (A/B/C/D) ごとの移動力 (battle sail と full sail の2種)
- Hull (船体)、Crew (乗員、複数セクションに分割)、Gun (左右舷側砲、bow/stern に細分)、Carronade (カロネード砲)、Rigging (索具・帆、速度段階ごとのセクション)、Reloads (装填回数、全30回)
- 移動プロット欄 (各ターンの動きを秘匿で記入)、ロード欄 (弾種、秘匿で記入)

**設計意図**: ログの損耗欄を削り消すことで、累積ダメージを視覚的に管理し、速度・火力・戦闘能力の低下が一目で分かる仕組み。

## 2. 移動 (Movement)

### 2.1 基本コンセプト
- 全艦の移動は**同時解決**。プレイヤーは各ターン、ログの移動欄に命令を秘匿で書き込む
- 命令実行フェイズで一斉に展開し、衝突が発生すれば1ヘクスずつ遡って解決

### 2.2 風向態勢 (Wind Attitude)
艦の針路と風向の関係を A/B/C/D の4態勢に分類:
- **A (追い風)**: 最大速度
- **B (横風寄り)**: 中間
- **C (向かい風寄り、close-hauled)**: 微速
- **D (真逆風、in irons)**: 移動不可 (艦は即停止し、そのターン以後移動・旋回できない)

### 2.3 移動力決定の流れ
1. 艦の **Battle Sail Speed (戦闘帆速度、3または4)** とターン開始時の風向態勢から、その艦の移動力 (ヘクス数) を算出
2. 各態勢ごとに移動可能ヘクス数の上限が別途存在 (例: 態勢Aでは4、態勢Cでは1)
3. 態勢変更後は、その新態勢の上限までしか進めない (重要)
4. 旋回は60度単位、1ヘクスにつき1回まで、1回の旋回で1移動力を消費
5. 艦には **Turning Ability (旋回回数上限)** がある (カウンタ記載)

### 2.4 全開帆 (Full Sails)
- Advanced Gameで導入。Full Sail Speed (5/6/7) により最大移動力が増大
- 代償:
  - 射撃時 Hit Table に-1(帆が不安定な射撃台となる)
  - 索具 (rigging) への被弾時、追加R (rigging) ダメージを受ける
  - 索具第1セクションを失うと即 battle sail に戻り、修理するまで再度上げられない
- 帆装変更 (BS ↔ FS) は Movement Notation Phase で宣言し、**Men in the Rigging (MIR)** として乗員1セクションを専任させる必要がある (射撃・接舷能力低下)

### 2.5 ドリフト (漂流 Drifting)
- 2ターン連続で bow hex が動かなかった場合、強制的に風下へ流される
- 大型 (SOL, クラス1-2) は2ターンに1ヘクス、小型は1ターンに1ヘクス
- 索具を全損 (dismasted) した艦はクラスに応じた待機ターン数後にしか60度旋回できない
- 索具・接舷で拘束された艦は「その場旋回」も不可、漂流のみ

### 2.6 バッキング (Backing Sails)
移動プロット中に「B」記号で1移動力を消費して**その場停止**。衝突回避や距離調整に使用。

### 2.7 衝突 (Collision) と索具絡み (Fouling)
- 同ヘクスに2隻が同時到達すると衝突
- どちらの艦がヘックスに残るかは優先順位規則 (先着優先、ダイスなど) で判定
- 衝突時に2d6でFouling Tableを参照、索具が絡むと両艦は動けなくなる
- 修正子の例:
  - 乗員品質差 (elite -2、crack -1、green +1、poor +2)
  - 片方が停止・漂流中: -4
  - 3ヘクス以上の速度: +1
  - full sail: +1/隻
  - 無修正12は必ず索具絡み

### 2.8 錨泊 (Anchoring)
- 通常錨 (A)、艦首錨 (AB)、艦尾錨 (AS) の3種
- 錨泊宣言は1ターン遅れで発効
- スプリング (springs) を使った艦首/艦尾錨は、錨を軸に最大120度旋回可能
- 錨泊艦は射撃命中率にボーナス (射撃台が安定)
- 揚錨 (up-anchor) は2連続ターン宣言、その間は射撃・修理・白兵戦不可

### 2.9 座礁 (Running Aground)
- 艦には **Depth Value (喫水)** があり、水深≤喫水のヘクスに進入すると座礁
- 座礁時、即1d6して離礁難度を決定。以降毎ターン Reload Phase でロールし、初期値より高い出目で離礁
- 砲を海中に投棄することで離礁判定に+1可 (1ターン1門、永久損失)
- 陸ヘクス侵入は艦破壊

## 3. 風・帆 (Wind and Sails)

### 3.1 風向 (Wind Direction)
- マップには風向ヘックスが1つ存在し、矢印で現在の風向 (1〜6のいずれか) を示す
- マップの各辺は風向番号でラベリングされている

### 3.2 風速 (Wind Velocity)
8段階: (0) Becalmed (無風) / (1) Light / (2) Moderate / (3) Normal / (4) Heavy / (5) Gale (強風) / (6) Storm (嵐) / (7) Hurricane (ハリケーン)

### 3.3 風変化判定 (Wind Change)
- 各シナリオに **Wind Change Number** が設定されている
- 毎ターンの Wind Phase で2d6を振り、合計が Wind Change Number と一致すれば変化発生
- 変化発生時はさらに1d6×2回振り、Wind Direction Changes Table と Wind Velocity Changes Table を参照

### 3.4 風速による効果
- Gale → Storm に変化: 全艦 full sail 解除、full sail中の艦は索具被害 (1d6の半分を切り上げ)
- Storm → Hurricane: ゲーム即終了 (得点リードで勝敗決定)
- Light → Becalmed: 旋回以外の移動不能
- Gale時: クラス1/5/6は Hit Table -1
- Storm時: クラス1/4は -2、クラス2/3は -1

### 3.5 風遮蔽 (Blocked Wind)
風上に隣接艦があり、その艦が自艦と風の間に位置すると帆が「遮蔽」され移動力減:
- 遮蔽艦がFSなら被遮蔽艦は-2ヘクス/ターン、遮蔽艦がBSで索具半分以上残なら-1ヘクス
- 艦サイズ差 (砲門数30/60差) で減速量補正

## 4. 砲撃 (Gunnery / Combat)

### 4.1 射撃の単位
- 射撃は**舷側斉射 (broadside)** 単位。各艦は左舷 (port/L) と右舷 (starboard/R) の2舷を持つ
- 射程は最大10ヘクス
- **Carronade (カロネード砲)**: 2ヘクス以内でのみ使用可。通常の舷側砲数に加算
- 射撃は同時解決。命中結果はフェイズ終了時に一括記入

### 4.2 射界 (Field of Fire)
**Basic Game**: 各舷側は「play」と呼ばれる扇形射界を持ち、その内の**最も近い敵艦**のみを標的にできる。最も近いのが陸・友軍・降伏艦・廃艦なら射線遮蔽で射撃不可。

**Advanced Game (射界細分)**: 舷側砲を bow section と stern section に分割し、射界を5つのフィールドに分割:
- Field 1/2/3: 全舷側斉射可能な領域
- Field 4: stern section のみ射撃可 (斜め後方)
- Field 5: bow section のみ射撃可 (斜め前方)
- 「同じフィールド内で最も近い艦」優先ルールがフィールドごとに独立適用。friendly fireリスクで射線遮蔽判定される

### 4.3 命中判定の流れ

射撃解決は以下の手順で行う:

1. **射程 (Range) を測定**: bow/stern どちらかまでの最短ヘクス距離
2. **砲数カウント**: 発射する舷側の生存砲マス数 (damaged分除外、射程2以内ならカロネード加算)
3. **Hit Determination Table (HDT) 参照**: 砲数 × 射程 のクロス参照で基本 Hit Table 番号 (0〜10) を取得
4. **修正子を合算** (後述)
5. **Hull または Rigging を選択**: 射程6ヘクス以上なら索具狙いに強制
6. 対応する Hit Table で**1d6**を振り、ダイス目と行をクロス参照して損害内容を取得
7. 損害は H (Hull)、G (Gun)、C (Crew)、R (Rigging) の4種で記載される

### 4.4 命中修正子 (HDT Modifiers)
- **乗員品質**: elite/crack で+、green/poor で- (砲数帯ごとに幅が変わる)
- **Rake (縦射)**: 艦首または艦尾方向から射撃した場合に大幅ボーナス。Advanced では艦尾方向に+1追加
- **乗員損失**: 完全に失われた乗員セクション1つにつき-1
- **初弾斉射 (Initial Broadside)**: 各舷の初撃のみボーナス (ターゲットなしで発射すると消失)
- **拿捕艦**: -2
- **Men in the Rigging**: -1 (crew section down扱い)
- **Full Sail射撃**: -1
- **錨泊**: ボーナスあり
- **Low Powder**: ペナルティ (後述)

### 4.5 Rake (縦射) 条件
- 対象艦の bow hex または stern hex が指している方向に位置し、その向きから射撃する場合にRake成立
- 距離1ヘクスで真正面/真後ろに付けた場合、敵が反撃可能でもRake認定 (例外)
- **射程5ヘクス以内でのみRake可能** (Advanced)
- Rakeは砲数と射程のクロスで得られる括弧内の Hit Table を使用

### 4.6 弾種 (Ammunition)
Advanced Gameで4種類使用可:

| 弾種 | 最大射程 | Hit Table | 装填 | 特徴 |
|---|---|---|---|---|
| **Roundshot (丸弾)** | 10 | 標準 | 1ターン | 汎用弾 |
| **Chainshot (鎖弾)** | 4 | +修正 | 1ターン | 索具専用、H/G命中は無効化。英国軍は使用不可 |
| **Doubleshot (2重弾)** | 1 | +修正 | 通常2ターン、乗員3セクション残存時は1ターン | 至近距離で強力。射程超過なら roundshot として発射可 |
| **Grapeshot (散弾)** | 1 | 表使用せず、修正後Hit Table番号 = 乗員撃破数 | 1ターン | 乗員専用、白兵戦2ターン目以降は使用不可。装填済 roundshot の直前に切り替え可 |

### 4.7 装填管理 (Reload)
- 各艦は **30発の装填枠** を持ち、1発装填ごとに1マス消費 (doubleshotは2マス)
- 使い切ると **Low Powder** 状態。射撃に-修正、さらに1の目で火災発生チェック (2d6=12で火災発生)
- 1ターンに1舷側のみ装填可能。両舷同時装填は不可

### 4.8 臼砲船 (Bomb Ketch)
- 錨泊中のみ臼砲発射可、障害物を越える高角射撃
- 装填4ターン、射程3-10ヘクス (最小3以上)
- 臼砲は Hit Table 2固定、ただし全ダメージ**3倍**
- 標的は陸上目標または停止中の艦のみ

## 5. 接舷・白兵戦 (Grappling and Melee)

### 5.1 接舷 (Grappling)
隣接艦に鉤縄で結縛:
- 2d6で Grappling Table を参照
- 修正子:
  - 乗員品質差 (敵艦接舷のみ)
  - 友軍同士なら +4 (ただし要ダイス、自動成功ではない)
  - 片方停止なら +1、両方停止 +2
  - 対象艦の移動力 (1超過分につき -1)
  - 風速1 (light) で +1、風速5 (gale) -1、風速6 (storm) -2
- 無修正12は常に成功
- 接舷した艦は移動・旋回不可、漂流のみ
- 乗員1セクションを使用 (白兵戦、水漏れ対応等に割り当て済は不可)

### 5.2 離舷 (Ungrappling)
- 2d6でUngrappling Table
- crack +1、elite +2、白兵戦中 **-6** (非常に離脱困難)、風速6で +1
- 無修正12は常に成功
- 友軍同士なら自動離脱
- 1接舷につき1試行。複数接舷されている場合は全て成功させないと自由にならない

### 5.3 接舷部隊 (Boarding Party) の形成
乗員セクションを3種類に割り当て、Boarding Preparation Phase で秘匿記入:

| 種別 | 記号 | 機能 |
|---|---|---|
| **Offensive Boarding Party** | OBP | 敵艦攻撃。一度形成すると白兵戦が決着/離舷/相互同意で中止されるまで継続義務 |
| **Defensive Boarding Party** | DBP | 防御。OBPに攻撃された時のみ白兵戦発生。被Rake時-1 |
| **Transfer Boarding Party** | TBP | 友軍艦への乗員移動。隣接友軍艦なら接舷不要 |

- 最低番号の乗員セクションから順に割当義務
- 全セクションをBP投入した艦はそのターン射撃不可、次ターン移動不可
- 修理中の艦はOBP不可 (DBPは可、ただし修理中止)

### 5.4 白兵戦 (Melee) 手順
1. **Total Melee Strength (TMS) 計算**: 各乗員マス × 乗員品質とBP種別ごとの倍率 (Crew Melee Strength Table)
   - DBP はOBPより乗員マスあたり +2点 (Rake時は+1)
2. 双方1d6振り、Melee Resolution Table で TMS列 × ダイス目を参照し相手が失う乗員マス数を決定
3. 白兵戦は同時解決。1ターン最大3ラウンド
4. **TMS比で3:1以上の差**が付くと劣勢側が降伏 (surrender by melee)、艦は敵の **Prize Crew (勝利部隊)** に拿捕される
5. 3ラウンドで決着しなければ次ターン継続。接舷維持であれば戦闘続行が義務

### 5.5 拿捕処理 (Capture)
- 白兵戦で降伏した艦はOBP側艦の勝利部隊が乗り込み拿捕
- 他の降伏 (強打/火力/不動による降伏) 艦は OBP または TBP が置かれた時点で拿捕
- 拿捕艦の砲発射は Hit Table -2 (乗員損失修正は無視)
- Prize crew 1マスにつき元乗員6マスまで管理可。これを下回ると囚人が反乱、立場逆転
- 拿捕艦への被弾時、乗員損失は奇数回目は勝利部隊、偶数回目は元乗員から取る (累積)

## 6. 損害・クリティカル・降伏 (Damage, Critical Hits, Surrender)

### 6.1 損害記録
Hit Table結果は log の該当セクションに **X 印**で記入:

| 損害種別 | 効果 |
|---|---|
| **H (Hull, 船体)** | Hullマスを1つ削る |
| **G (Gun, 砲)** | 砲マスを1つ削る。射撃側に近い舷から削る |
| **C (Crew, 乗員)** | 乗員を最低番号セクションから削る |
| **R (Rigging, 索具)** | 索具を最高番号セクションから削る。1セクション全滅ごとに全態勢で移動力-1 |

### 6.2 降伏 (Surrender) 4パターン
1. **Strike (旗降ろし)**: Hullマス全損 → 艦は沈没寸前、戦闘・航行不能。5ヘクス以内に友軍が来れば解除される (ダメージは残ったまま)
2. **Firepower (火力降伏)**: Gunマス全損かつ10ヘクス以内に同等以上クラスの友軍艦なし → 接近して舷側斉射した敵艦に降伏
3. **Immobility (不動降伏)**: Riggingマス全損かつ10ヘクス以内に同等以上友軍なし → Rakeを受けた時点で降伏
4. **Melee (白兵降伏)**: TMS 3:1で勝敗決着

### 6.3 沈没・爆発 (Sink / Explode) - Advanced
- Hull全損で Strike発生時、Destroyed Hull Table を参照して「降伏のみ/沈没予定/爆発予定」を判定
- 沈没・爆発予定艦は毎ターン Unfouling Phase で1d6、6の目で実行
- 爆発時、隣接艦は爆発元の砲数から算出される Hit Table (最大10) で索具被害、さらに火災クリティカル2Hチェック

### 6.4 クリティカルヒット (Critical Hit) - Advanced
- 各 Hit Table の**ダイス目6の結果**にはアスタリスク = クリティカル可能性
- 6が出た場合、2d6で Critical Hit Table 参照 (Hull側とRigging側で別表)
- 重篤な結果は「Hit Table番号以下の qualifying roll」が必要 (低テーブルでは軽微な結果しか発動しない)
- クリティカル効果は**Combat Phase終了時**に他ダメージ後に適用 (累積)
- 艦首斜桁折損、舵故障、艦長戦死、士気崩壊、弾薬庫誘爆、浸水、火災、帆柱倒壊 など

### 6.5 帆柱損失 (Loss of Rigging / Mast)
- 索具セクション全滅時に1d6:
  - 1: 帆柱が舷側に倒れかかる (さらに1d6でどちらの舷か決定)
  - 2-6: 清く折損
- 舷側に倒れた帆柱: 片舷射撃不能、旋回不能、速度-1、Unfouling判定で切り離す必要あり
- 全索具喪失艦 (dismasted) は砲6マス以下で -1HT、7マス以上で -2HT の射撃ペナルティ

### 6.6 浸水 (Waterline Damage)
クリティカルの一種。乗員セクションを排水対応に割くと、他行動 (白兵・射撃) 不可となる。

## 7. 手番進行 (Sequence of Play)

### 7.1 Basic Game (8フェイズ)
1. Unfouling Phase (索具絡み解除試行)
2. Movement Notation Phase (移動命令記入、秘匿)
3. Movement Execution Phase (同時移動実行、衝突処理)
4. Grappling and Ungrappling Phase (接舷・離舷試行)
5. Boarding Preparation Phase (接舷部隊記入、秘匿)
6. Combat Phase (射撃解決)
7. Melee Phase (白兵戦解決)
8. Load Phase (装填)

### 7.2 Advanced Game (10フェイズ)
1. **Wind Phase** (風変化判定、追加)
2. Unfouling Phase (沈没・爆発ロール含む)
3. Movement Notation Phase (修理・錨・帆装変更もここで記入)
4. Movement Execution Phase
5. Grappling and Ungrappling Phase
6. Boarding Preparation Phase
7. Combat Phase (沈没・爆発判定含む)
8. Melee Phase
9. Reload Phase (修理完了処理もここ)
10. **Full Sail Phase** (帆装変更の反映、追加)

### 7.3 設計意図
- 移動・装填・接舷部隊がすべて**秘匿書込み → 同時解決**という設計。プレイヤー双方が同時に判断するため、心理戦・予測要素が強い
- 修理・錨・帆装変更などもすべて Movement Notation Phase で事前宣言 (「後から決められない」ことで、判断の質が問われる)

## 8. その他特殊ルール (Special Rules)

### 8.1 修理 (Repairs) - Advanced
- RH (Hull修理) / RG (Gun修理) / RR (Rigging修理) の3種
- 事前宣言後、3ターンで2マス回復 (連続でなくてよい)
- 修理中は射撃・移動での旋回・帆装変更・錨操作・接舷戦闘すべて不可
- 条件違反で修理は無効化、射撃も不可
- 最低2乗員セクション必要

### 8.2 曳航 (Towing)
- 友軍艦を曳航可、曳航艦は速度-1〜-2ヘクス
- 風速2-4でのみ曳航維持可能 (適切な風向必須)
- クリティカル5H (錨索切断) で曳航索も切れる可能性あり

### 8.3 測深 (Casting the Lead)
1ヘクス以下の移動なら「測深中」とみなされ、隣接ヘクスの水深情報を敵プレイヤーに質問できる。

### 8.4 Fore and Aft Rigging (縦帆装)
縦帆装艦 (sloop等) は態勢Dでも2ターンに1ヘクス進める特例。逆風性能が高い帆装を表現。

### 8.5 Copper Bottom / Scurvy (銅板被覆・壊血病)
3ヶ月以上出動かつ銅板なし艦は full sail時 -1ヘクス。新規配属艦は乗員増量扱い。いずれもオプション。

### 8.6 Command Lag (指揮遅延) - マルチプレイヤー
- 6隻以下の戦隊単位で編成、旗艦 (flag ship) を指定
- 旗艦から10ヘクス超の艦は**3ターン先まで移動命令を事前プロット**必要 (命令遅延シミュレーション)
- 旗艦降伏で5ターン先まで事前プロット強制

### 8.7 Multi-Player Communication
マルチプレイ時、同艦隊内通信は Movement Notation Phase で書面のみ、1艦1メッセージ/ターン。

### 8.8 Variable Wind (可変風)
シナリオ設定の風を放棄し、3回ダイスを振って風向・風速・変化番号をランダム決定。

### 8.9 Timed Moves
命令記入に制限時間 (単艦1分、3隻まで2分、6隻追加ごと+3分)。時間切れ艦は直進命令。

## 10. 帆船戦ゲーム実装上の要点

1. **秘匿同時記入 → 同時解決**が中核。UIは命令記入フェイズと実行フェイズを明確に分離する設計が必要
2. **風向態勢 (A/B/C/D)** は自艦向きと風向の相対角度から機械的に算出可能。角度→態勢→移動力テーブル参照の3段階
3. **Rake判定**は対象艦の bow/stern hex 指向ベクトルと発射艦位置のヘクス関係で判定。5ヘクス以内制限
4. **射界 (Field of Fire)** は Advanced では5分割。各フィールドごとに独立した「最接近対象」判定が必要
5. **ログ管理**: 艦ごとに Hull/Gun/Crew/Rigging/Reload の複数配列状態を持ち、各命中で該当配列を削る。艦性能 (移動・命中) は残存数から動的算出
6. **クリティカル・沈没・爆発**は遅延処理 (Combat Phase終了後適用、毎ターン6の目待ち)。状態マーカーが必須
7. **白兵戦**は複数ターンに渡る継続状態。TMS計算は毎ラウンド再計算、3:1で決着
8. **修理・接舷・帆装変更**は相互排他ルールが複雑。フェイズ遷移時の状態バリデーションが肝要
9. **弾種切替**は装填サイクル制約あり。ただしroundshot→grapeshot、doubleshot→roundshot 等の直前切替ルールは実戦的な柔軟性を担保

以上がWS&IM v1.2b の主要メカニクスの構造的要約である。実装参考として、各判定のダイス・テーブル構造、フェイズ順序、状態遷移ルールを中心に整理した。
